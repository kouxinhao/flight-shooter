# 退出登录返回开始界面 Spec

## Why

当前退出登录后仅清除了用户状态和隐藏用户标签，游戏画面仍停留在当前界面（菜单/游戏中/暂停中），用户需要手动刷新才能回到开始界面，体验不连贯。

## What Changes

- 点击"退出"按钮后，游戏状态强制切换到菜单界面（开始界面）
- 如果是游戏中或暂停状态，先重置游戏再回到菜单
- 确保退出后所有游戏数据（敌人、子弹、道具等）被清除

## Impact

- 影响文件：`script.js` — 修改 `onLogout()` 函数
- 不影响登录流程、不影响服务端数据、不影响未登录用户的游戏体验

## ADDED Requirements

### Requirement: 退出登录重置游戏状态

#### Scenario: 游戏中退出
- **WHEN** 用户在游戏进行中点击"退出"按钮
- **THEN** 游戏立即停止，敌人/子弹/道具等实体全部清除
- **THEN** 游戏画面回到菜单界面（开始界面）

#### Scenario: 暂停中退出
- **WHEN** 用户在暂停状态点击"退出"按钮
- **THEN** 暂停状态解除，游戏回到菜单界面

#### Scenario: 菜单中退出
- **WHEN** 用户在菜单界面点击"退出"按钮
- **THEN** 菜单界面保持不变（已处于开始界面）

## MODIFIED Requirements

### Requirement: onLogout 函数
- 修改：在清除用户状态后，增加 `gameState = 'menu'` 和游戏重置逻辑
