# 游戏体验增强 Spec

## Why
当前飞行射击游戏缺少开始界面、音效系统和移动端支持，限制了游戏的可玩性和受众范围。本次优化将补齐这三块短板。

## What Changes
- 增加游戏开始界面（标题、操作提示、开始按钮）
- 使用 Web Audio API 添加音效系统（射击音效、爆炸音效、背景音乐）
- 移动端适配：触屏控制（虚拟摇杆 + 自动射击）

## Impact
- 修改文件：`index.html`、`style.css`、`script.js`
- 无需新增外部依赖，音效通过 Web Audio API 程序化生成

## ADDED Requirements

### Requirement: 游戏开始界面
系统 SHALL 提供开始界面，在游戏加载时显示：
- 游戏标题 "飞行射击游戏"
- 操作提示文字（方向键/WASD 移动，空格射击）
- 移动端显示触屏操作提示
- "点击开始" 按钮或按任意键进入游戏

#### Scenario: 打开游戏
- **WHEN** 用户打开游戏页面
- **THEN** 显示开始界面，等待用户操作后进入游戏

### Requirement: 音效系统
系统 SHALL 使用 Web Audio API 程序化生成音效：
- 射击音效：短促高频音
- 击中敌机音效：爆炸音
- 玩家死亡音效：低沉爆炸音
- 背景音乐：简单的循环低音节奏
- 音效不影响游戏性能，可开关

#### Scenario: 玩家射击
- **WHEN** 玩家按下空格键发射子弹
- **THEN** 播放射击音效

#### Scenario: 击中敌机
- **WHEN** 玩家子弹击中敌机
- **THEN** 播放爆炸音效，得分增加

### Requirement: 移动端触屏控制
系统 SHALL 支持移动端触屏操作：
- 屏幕左侧区域：触摸并滑动控制战机移动（虚拟摇杆）
- 自动射击：触摸屏幕时持续自动发射子弹
- 响应式画布：根据屏幕尺寸缩放画布
- 禁止页面滚动，防止误触

#### Scenario: 触屏移动
- **WHEN** 用户在屏幕左侧触摸并滑动
- **THEN** 战机跟随触摸位置移动

#### Scenario: 触屏射击
- **WHEN** 用户触摸屏幕任意位置
- **THEN** 战机持续自动射击

## MODIFIED Requirements

### Requirement: 游戏状态管理（修改）
系统 SHALL 管理三个游戏状态：
- `menu` — 开始界面
- `playing` — 游戏进行中
- `over` — 游戏结束

#### Scenario: 状态流转
- **WHEN** 用户在菜单界面点击/按键
- **THEN** 状态切换为 `playing`
- **WHEN** 玩家死亡
- **THEN** 状态切换为 `over`
- **WHEN** 用户在结束界面点击/按键
- **THEN** 状态切换为 `menu`（回到开始界面）
