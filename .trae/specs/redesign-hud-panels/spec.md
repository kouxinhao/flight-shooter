# 酷炫 HUD 面板重设计 Spec

## 为什么

当前 HP 面板和武器能源面板样式保守、尺寸小、缺乏视觉冲击力。HP 分段格太矮小、能量条太薄、整体像简陋的调试面板而非游戏 HUD。需要彻底重做，走"酷炫"路线：大、发光、动态、有层次感。

## 变更内容

### HP 面板（重塑）
- 面板尺寸：180px × 64px（加高，增加视觉权重）
- 去掉旧的分段格（短而小），改为 **5 个圆形 HP 宝珠**，每颗代表 1 点 HP
  - 填充态：红橙渐变发光球体，带白色高光点，微微脉动
  - 空态：暗红色半透明圆圈，几乎不可见
  - 受伤动画：失去的宝珠闪白爆发（爆炸缩放效果）
- 左侧：发光 ❤ 图标 + "HP" 标签竖排
- 右侧：超大 HP 数字（"5" 字号 28px）
- 护盾：所有宝珠外围半透明蓝色旋转光环线
- 低血量：宝珠边框红色脉动、心跳动画

### 武器能源面板（重塑）
- 面板尺寸：180px × 48px（加高）
- 替换旧薄条为 **厚能量槽**（18px 高），带流动光泽动画
  - 渐变：青色 → 蓝紫 → 亮青（流动）
  - 100% 时全槽脉冲发光
  - 接近 0% 时红色警告
- 武器等级改为发光徽章，嵌入在能量槽右端
- 能量百分比数字（14px monospace）放在能量槽上方

### 两面板共享设计语言
- 背景：半透明黑色 → 改为更深邃的渐变玻璃质感
- 边框：彩色发光边框（HP 红色系、能源青蓝系）
- 字体：数值全部加大，带发光 shadow
- 动画：数值变化时有平滑过渡

## 影响

- 受影响模块：HUD 覆盖层
- 受影响代码：
  - `index.html` — HP 和能源面板 DOM 结构重写
  - `style.css` — HP 宝珠、能量槽、护盾环、动画全部新样式
  - `script.js` — `updateHUDOverlay()` 更新逻辑适配新结构

## 新增需求

### 需求：圆形 HP 宝珠系统

#### 场景：HP 满值
- **给定** `player.hp = 5, maxHp = 5`
- **当** 游戏处于 playing/paused/over
- **那么** 显示 5 颗发光宝珠，每颗 16px 直径，红橙渐变，带白色高光点

#### 场景：HP 减少
- **给定** 玩家受伤
- **当** `updateHUDOverlay()` 检测到 hp 减少
- **那么** 减少的那颗宝珠执行 300ms 白色爆发缩放动画后变暗

#### 场景：HP 为空
- **给定** `hp = 0`
- **那么** 5 颗宝珠全部暗色半透明，面板显示低血量警告

### 需求：厚能量槽系统

#### 场景：能量变化
- **给定** `energyRatio = 0.75`
- **当** 更新 HUD
- **那么** 能量槽填充 75%，青色渐变，流动光泽动画持续运行

#### 场景：能量满
- **给定** `energyRatio >= 1.0`
- **当** 更新 HUD
- **那么** 能量槽额外发光脉冲

#### 场景：武器等级升级
- **给定** `weaponLevel` 变化
- **当** 更新 HUD
- **那么** 等级徽章显示新等级，触发黄色闪烁动画
