# 增强 HP 面板视觉设计 Spec

## 为什么

当前 HP 生命值面板下半部分（进度条区域）视觉上空荡荡的，只是一个薄条配上大面积黑色背景。这浪费了面板空间，也让生命值状态不够直观和酷炫。玩家对生命值的感知应该更强烈——尤其在战斗中。

## 变更内容

- **HP 面板尺寸加大**：高度从 36px → 56px，给下半部分更多空间
- **分段式 HP 条**：将连续进度条改为每点 HP 一个独立分段格（5 格对应 maxHp=5），每个格子有精致填充+边框+光泽
- **护盾叠加显示**：玩家获得护盾时，HP 条上方叠加蓝色半透明护盾层
- **HP 数值视觉增强**：数字放大+发光，低血量时数字变为红色闪烁
- **装饰元素**：HP 条末端添加小型心形图标，低血量时动态跳动
- **受伤动画升级**：受伤闪白效果保留并增强（分段格逐个闪烁）

## 影响

- 受影响模块：HUD 覆盖层
- 受影响代码：
  - `index.html` — HP 面板 DOM 结构调整
  - `style.css` — HP 面板新样式（分段条、护盾叠加、动画）
  - `script.js` — `updateHUDOverlay()` 更新逻辑适配新结构

## 新增需求

### 需求：分段式 HP 条

系统应使用独立分段格显示每点生命值，而非单一连续进度条。

#### 场景：HP 满值时
- **给定** 玩家 `hp = 5, maxHp = 5`
- **当** 游戏处于 playing/paused/over 状态
- **那么** HP 面板显示 5 个满填充分段格，每个格子从红渐变到橙

#### 场景：HP 减少时
- **给定** 玩家 `hp = 3, maxHp = 5`
- **当** 玩家受伤
- **那么** 前 3 格满填充，第 4 格起为空框，当前减少的 HP 所在格执行闪自动画

#### 场景：低血量警告
- **给定** 玩家 `hp = 1, maxHp = 5`
- **当** `hpRatio < 0.3`
- **那么** 空框变为红色脉动边框，数字闪烁

### 需求：护盾叠加显示

#### 场景：玩家拥有护盾
- **给定** `player.shield = true`
- **当** 更新 HUD
- **那么** HP 条上方叠加一层半透明蓝色护盾覆盖层

### 需求：受伤动画

#### 场景：HP 减少时
- **给定** HP 值减少
- **当** `updateHUDOverlay()` 检测到 `player.hp < _prevHp`
- **那么** 对应分段格触发 200ms 白色闪烁动画

## 修改的需求

### 需求：HP 面板尺寸
**之前**: 高度 36px，薄条 8px
**之后**: 高度 56px，分段格区域占据下半部分
